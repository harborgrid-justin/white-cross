# NestJS Best Practices Audit Plan

## Agent Assignments

### 1. nestjs-controllers-architect
**Focus**: Controllers, routing, request handling, HTTP decorators
**Scope**: All `*.controller.ts` files in backend/src/
**Key Areas**:
- Route structure and naming conventions
- HTTP method usage
- Request/Response handling
- Controller organization
- Decorator usage

### 2. nestjs-providers-architect
**Focus**: Services, dependency injection, business logic
**Scope**: All `*.service.ts` files in backend/src/
**Key Areas**:
- Service patterns and structure
- Dependency injection practices
- Provider organization
- Business logic separation

### 3. nestjs-configuration-architect
**Focus**: Configuration management, environment variables
**Scope**: Configuration files, .env usage, config modules
**Key Areas**:
- ConfigModule setup
- Environment variable validation
- Configuration patterns

### 4. nestjs-security-architect
**Focus**: Authentication, authorization, security best practices
**Scope**: Guards, auth modules, security middleware
**Key Areas**:
- Authentication mechanisms
- Authorization guards
- Security headers
- Data validation
- Input sanitization

### 5. nestjs-testing-architect
**Focus**: Testing coverage, test patterns, test quality
**Scope**: All `*.spec.ts` files
**Key Areas**:
- Unit test coverage
- Integration tests
- E2E tests
- Test patterns and mocking

### 6. database-architect
**Focus**: Database design, schema, query optimization
**Scope**: Entities, repositories, database modules
**Key Areas**:
- Entity relationships
- Database schema design
- Query optimization
- Migration practices

### 7. api-architect
**Focus**: API design, REST conventions, versioning
**Scope**: API endpoints, DTOs, response structures
**Key Areas**:
- REST API design
- DTO patterns
- Response formatting
- Error handling
- API versioning

### 8. typescript-architect
**Focus**: TypeScript best practices, type safety, code quality
**Scope**: All TypeScript files
**Key Areas**:
- Type safety
- Interface design
- Generic usage
- Code organization
- JSDoc documentation
